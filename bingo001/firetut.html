üé≤ Project Overview

This is a fully functional real-time multiplayer Bingo game built with vanilla HTML/CSS/JS and Firebase. Players sign in with Google, create or join lobbies via unique IDs, receive unique randomized 5x5 boards (numbers 1-25), and play in real-time. The host calls numbers, boards update instantly across players, and anyone can claim Bingo when they have a row/column/diagonal.

Key Features:
‚Ä¢ Google Auth with session persistence
‚Ä¢ Real-time Firestore sync via onSnapshot
‚Ä¢ Unique boards per player
‚Ä¢ Client-side Bingo validation
‚Ä¢ Responsive dark theme UI
‚Ä¢ Lobby listing & direct join by ID

Files: [`index.html`](index.html) (main lobby), [`lobby.html`](lobby.html) (game), [`script.js`](script.js) (logic).

‚öôÔ∏è Firebase Setup

  1. Create project at console.firebase.google.com

üîê Authentication

Uses Firebase Auth v12.6 with Google provider and redirect (session persistence). See [`script.js`](script.js:2-9) imports, [`script.js`](script.js:37-39) init.

window.loginGoogle = async () => {
  setPersistence(auth, browserSessionPersistence);
  await signInWithRedirect(auth, googleProvider);
};

onAuthStateChanged handles state: show/hide buttons, list games [`script.js`](script.js:453).

üìä Firestore Logic

Lobbies collection: one doc per game. Structure:

{
  status: "waiting",
  hostUID: "uid",
  players: { uid: {displayName, board: [1..25]} },
  calledNumbers: [],
  winnerUID: null
}

Ops: addDoc(create), updateDoc(join/call/claim), onSnapshot(realtime), query(waiting lobbies) [`script.js`](script.js:13-20).

üîß Troubleshooting

  • Auth fail: Check authorized domains (localhost)
  • Permission denied: Firestore rules or unauth
  • No realtime: Wrong lobbyId or snapshot error
  • Board not unique: Shuffle logic
Console: Check Firebase errors. Test: Create lobby, join incognito.

üéÆ How to Play
